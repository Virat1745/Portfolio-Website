CHALLENGES

When I started building this website, I thought it would be simple—just HTML and CSS, right?


But very quickly, I learned that designing and hosting a custom website from scratch is far from basic.


The first real challenge was deciding on a layout that actually felt like me.

I didn’t want a boring template or something bloated. I wanted a clean, minimal, and unique feel—something that truly represents who I am.


Then came the actual implementation. Getting HTML, CSS, JavaScript, and all the media to play nicely together took more trial and error than I expected.


Once that was done, I didn’t want it to just sit on my laptop. I wanted it live—accessible anywhere in the world.


That opened a whole new rabbit hole: domain registration, static IP setup, configuring Nginx, setting up HTTPS with Certbot, and finally hosting it all on my Raspberry Pi.


I had to deal with SSL errors, DNS propagation delays, port-forwarding complications, and a few late nights debugging why the site refused to load.


But slowly, piece by piece, it came together. Every problem taught me something new, and by the end of it, I didn’t just have a website—I had a project I was proud of.


LEARNINGS

Now, after completing this, I have a better understanding of:

Web & Networking

  • Responsive Design — viewport scaling, mobile-first layouts, accessibility.
  • Static Hosting — serving a fully static frontend with Nginx.
  • DNS Configuration — pointing a domain (e.g., viratportfolio.com) to a home-hosted server.
  • HTTPS with SSL — obtaining and renewing Let’s Encrypt certificates via Certbot.

Linux & Server Admin

  • Nginx Configuration — editing /etc/nginx/sites-available/ and enabling it properly.
  • File Management — organizing media, scripts, and assets in clean directories.
  • System Performance — tools like htop, df -h, and journalctl for monitoring.
  • Startup Services — configuring systemd to auto-start Nginx and other components on boot.

Design & Development

  • Minimal UI/UX — simplicity, spacing, contrast, and smooth navigation.
  • AOS Animations — integrating scroll-based effects.
  • Semantic HTML — proper tags for SEO and accessibility.
  • Deployment Mindset — testing, debugging, and thinking like a user, not just a coder.
