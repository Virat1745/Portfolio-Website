CHALLENGES

When I started building this private cloud,

I underestimated the challenges I would face along the way.

The first hurdle was setting up permissions on the Raspberry Pi to allow file uploads, and then forwarding those uploads to a USB drive.

Once I got that working, I thought—why not integrate this with my personal website?
And oh boy, that was tough—making Nginx and Flask work together was a challenge in itself.

After setting that up, I knew I couldn’t leave the upload page open to everyone. Anyone could upload malware, viruses, or even try to crash my Pi, hack my website, or get into my home network.

So, I implemented a secure login system—only authorized users can access the cloud upload page now.

Each problem pushed me to dig deeper, and with every roadblock, I gained sharper skills and greater confidence.


LEARNINGS

Now after completing this I have a better understanding of

Networking & Web:

Port Forwarding — configuring routers to allow external access to internal devices.
DNS Setup — mapping domain names to your Raspberry Pi's public IP.
DHCP Awareness — understanding static vs dynamic IPs, especially for your Pi.
Firewall Basics — likely through ufw or iptables to secure access.

Linux & Server Admin:

Systemd Services — possibly creating a service to run Flask/Nginx on boot.
Directory Structure — understanding /etc/nginx/, /var/www/, or /media/usb.
Mounting Drives — configuring /etc/fstab or using mount to auto-mount USBs.
Using Terminal Commands — ls, cp, mv, nano, systemctl, top, df, etc.


Security Concepts:

Authentication — building login systems to restrict access.
Sanitizing Input — preventing malicious file names or types.
Basic Threat Modeling — thinking like an attacker (what could go wrong?).
HTTPS Setup with Certbot/SSL — handling Let’s Encrypt or self-signed certs.
